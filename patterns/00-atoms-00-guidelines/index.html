Code Guidelines

1) Class names

.breadcrumbs (standaard)
.breadcrumbs-list (subitem)
.breadcrumbs-item (subitem)
.breadcrumbs--footer (variant op standaard | gebruik: class = "breadcrumbs breadcrumbs--footer")

.pageHeader (standaard meerdere woorden met camelCase)
.pageHeader-link
.pageHeader--home

States

.is-opened
.is-closed
.is-animating
.is-selected
.is-disabled

Javascript / Angular

.js-openClose / .js-openClose-content
.js-accordion / .js-accordion--mobile

Angular Animation Classes

.ngAnimate-height
.ngAnimate-scale
.ngAnimate-opacity

2) Mixin names

Altijd camelCase!

@mixin accShow
@mixin showText

3) Var names

Altijd camelCase!

$lightBlue
%addSidePadding
$blockMargin

4) Units

rem - als ie 'vast' moet staan (alleen als de hele pagina schaalt)
em - als ie mee moet schalen met het element zelf (bijv. "personeel >")
% - met name voor breedtes (bijv. blokken)

Viewport

vw - breedte van browserscherm = 100vw
vh - hoogte van het browserscherm = 100vh
vmin
vmax

Voor de definitie van borders kan/mag wel gebruik worden gemaakt van pixels, omdat het strakke, voorspelbare lijnen moeten zijn.
